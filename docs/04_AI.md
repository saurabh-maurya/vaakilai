## 2. AI Architecture

### 2.1 High-Level AI Pipeline

```
┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│  User Query  │────▶│  AI Gateway   │────▶│  Intent Router   │
└─────────────┘     └──────────────┘     └────────┬────────┘
                                                   │
                    ┌──────────────────────────────┼──────────────────┐
                    │                              │                  │
              ┌─────▼─────┐              ┌────────▼──────┐   ┌──────▼──────┐
              │Consultation│              │  Document Gen  │   │  Research   │
              │  Engine    │              │   Pipeline     │   │  Pipeline   │
              └─────┬─────┘              └────────┬──────┘   └──────┬──────┘
                    │                              │                  │
              ┌─────▼──────────────────────────────▼──────────────────▼─────┐
              │                    RAG Layer                                │
              │   Query → Embed → Vector Search → Retrieve → Rerank         │
              └─────┬───────────────────────────────────────────────────────┘
                    │
              ┌─────▼─────┐     ┌──────────────┐     ┌─────────────────┐
              │ LLM Core  │────▶│  Guardrails   │────▶│  Response +     │
              │Claude/LLaMA│     │  & Citations  │     │  Citations      │
              └───────────┘     └──────────────┘     └─────────────────┘
```

### 2.2 RAG (Retrieval-Augmented Generation) Architecture

| Stage | Component | Details |
|-------|-----------|---------|
| Ingestion | Document parser | SC/HC judgments, statutes, regulations → chunked |
| Embedding | Legal embedding model | Fine-tuned on Indian legal text |
| Indexing | Pinecone / Weaviate | 4M+ judgment vectors + statute vectors |
| Retrieval | Hybrid search | Semantic + keyword (BM25) with jurisdiction filters |
| Reranking | Cross-encoder | Top-K reranking for relevance |
| Generation | Claude + LLaMA | Context-augmented response with citations |
| Validation | Citation checker | Verify cited cases exist and are current |

## 8. AI Safety & Responsible AI

### 8.1 Guardrails

| Guardrail | Implementation |
|-----------|----------------|
| Disclaimer enforcement | Every response includes AI disclaimer; blocked if missing |
| Hallucination prevention | RAG-only responses; uncited answers blocked |
| Citation validation | Post-generation check: cited cases must exist in corpus |
| Confidence threshold | Responses below 0.6 confidence → "consult a lawyer" redirect |
| High-stakes routing | Criminal, bail, adoption → human lawyer review queue |
| Bias auditing | Quarterly audits across gender, religion, caste, socioeconomic |
| Explainability | Log retrieval sources for every response; expose to lawyers in Pro |

### 8.2 High-Stakes Matter Policy

These topics **always** route to human lawyer review before delivery:

- Criminal matters (FIR, arrest, bail)
- Adoption & guardianship
- Domestic violence protective orders
- Property disputes involving eviction
- Any matter involving minors

### 8.3 AI Ethics Compliance

| Policy | Requirement |
|--------|-------------|
| Advocates Act compliance | AI is "information tool"; not "practice of law" |
| Source attribution | Every legal answer cites judgment or statute with hyperlink |
| No legal advice claim | UI + API always label output as "AI-generated guidance" |
| Data privacy | No user queries stored for training without explicit consent |
| Model transparency | Lawyers can inspect retrieval sources for any research output |
